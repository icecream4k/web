MintUI
组件库基础
    什么是组件库
  • 组件是企业为提升开发效率而开发,对于相关的样式及交互的封装,其提供了简单的调用方式和接口,让开发者可以便捷实用其提供的功能来完成业务需求。实现了一次封装,多次调用
    组件库的优势
  • 提升开发效率
  • 便于团队协同开发
  • 简化调试步骤
  • 提升项目的可维护性
    组件库的分类
  • 移动端的组件库
    • MintUI(饿了么团队开发)
    • VantUI(有赞团队开发)
    • cubeUI(滴滴团队开发)
  • 桌面端组件库
    • ElementUI(饿了么团队开发)
    • ATUI(凹凸团队开发)
    • viewUI(试图更新)
  • 移动端与桌面端的区别
    • 屏幕尺寸不同
    • 操作方式不同
    • 事件触发方式不同(ontouchstart,ontouchend)
    • 网络环境不同

使用Mint UI
    CSS组件
  • Header组件
    Header组件用于实现顶部导航,其语法结构是:
    <mt-header title='标题信息' fixed></mt-header>
    title属性用于设置标题信息,字符类型
    fixed属性用于控制是否固定在顶部,布尔类型
    <mt-header>子元素可以通过slot='left'或slot='right'控制子元素显示的位置
  • 顶部选项卡
    <mt-navbar fixed v-model="变量名称">
    <mt-tab-item id="当前选项卡ID">
    </mt-tab-item>
    </mt-navbar>
    说明：fixed属性用于控制顶部选项卡是否固定在页面顶部,boolean类型；
    v-model 指令绑定的变量类型为字符类型,其应该为当前被激活的选项卡的id
    可以子啊<mt-tab-item>中嵌套图像,并且为图像设置slot='icon'属性,此时图像将作为选项卡的图标
  • 在src/components目录下创建MintUI目录
  • 创建组件文件,书写相对应的标签--template,script,style
  • 在根组件或父组件文件内,导入创建的组件文件
  • 设置路由信息
  • 创建连接或绑定事件,以访问路由
    Reset.css
    每个浏览器对于HTML元素都有默认样式,但是这些默认样式在不同的浏览器中是不相同的, 为了保证所有浏览器对HTML元素默认样式是一致的,建议使用reset.css对HTML元素的默认样式是一致的, 建议使用 reset.css对HTML元素的默认样式进行重置
    Button组件
  • Button组件用于实现按钮,其语法结构是:
    <mt-button type='按钮样式' size='尺寸' icon='图标' disabled>
    ....
    </mt-button>
      按钮样式
    • default(默认)
    • primary(主要)
    • danger(危险)
      尺寸
    • small(小的)
    • normal(标准)
    • large(大的)
      图标
    • back(返回)
    • more(更多)
    • 自定义图标
      disabled属性用于控制按钮是否为禁用的状态,布尔类型,默认值为false
      可通过slot='icon'属性将图像作为按钮图标
        如果使用第三方图标,比如阿里图标库的图标
      • 1,解压到src/assets/styles目录
      • 2,在对应的组件中导入css文件
      • import ‘css路径’
      • 3,按钮上添加icon='iconfont icon-xxx'
      • 使用自定义字体图标时需要注意选择器的优先级


    表单组件
      Field组件
    • field组件用于实现表单控件,其语法结构是:
    • <mt-field type = '输入框的类型' lable='标签'
      placeholder = '占位内容' state='检测状态' v-model = '变量名称' :attr = '原生属性'
      readonly
      disabled
      disableClear
      >
      </mt-field>
    • 输入框的类型有 - type:
      text,单行文本框
      passsword,密码框
      textarea,多行文本框/域
      number,数字
      tel,电话
      url,URL地址
      date,日期
      datetime-local,日期和时间
    • 占位符 - placeholder
    • state属性表示输入框的状态,其值有:
      error - 表示错误
      success - 表示成功
      warning - 表示警告
    • :attr - 属性用于控制表单控件的原生属性,对象类型,
      如果是密码输入框,记得加入autocomplete:'off',防止浏览器自动填充密码
    • readonly - 属性用于控制表单空间是否为只读状态,布尔类型,默认为false
    • disabled - 不让输入
    • disableClear - 让输入框后面的叉消失
    • 如果为表单控件添加失去焦点、获取焦点的事件,其语法结构是
      <mt-field @blur.native.capture="事件名称"></mt-field>
      <mt-field @focus.native.capture="事件名称"></mt-field>
    -------------------
    • 复选框组件
      <mt-checklist title="标题"
      align="对齐方式(left|right)"
      :options="选项列表"
      v-modle="变量"
      ></mt-checklist>
      :options属性必须存在,options属性为array类型,既可以直接存储字符串,也可以存储对象
      v-model也必须存在,且数据类型必须为array类型
      如果为字符串的话,则为最后的选定值泽为选定的文本,如果是对象的话,最后的值是value中的数据
    • 单选框
      <mt-radio title="标题"
      align="对齐方式(left|right)"
      :options="选项列表"
      v-model="变量名称"
      >
      </mt-radio>
      :options属性必须存在,options属性为array类型,既可以直接存储字符串,也可以存储对象
      v-model也必须存在,且数据类型必须为String类型
    • 开关组件
      <mt-switch v-model="变量名称"
      disabled>
      </mt-switch>
      说明：v-model指令绑定的变量类型为boolean类型
    • 底部选项卡
    • <mt-tabbar v-model='变量名称' fixed>
      <mt-tab-item id="当前选项卡的ID">
      ..
      </mt-tab-item>
      </mt-tabbar>
      v-model指令绑定的变量类型为字符类型,其应该为当前被激活的底部选项卡的id
      fixed属性用于控制是否在页面的底部来固定选项卡,boolean类型


    JS组件
      toast组件
        toast组件用于显示短消息提示框,其语法结构是:
      • this.$toast('消息提示')
      • this.$toast({ message:'提示信息', position:'位置(top|middle|bottom)', duration:'持续时间(单位为毫秒,默认为3000)', iconClass:'字体图标类名称' })
